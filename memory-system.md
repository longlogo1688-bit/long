# 记忆连续性增强系统

## 系统说明
为野原定制的上下文连接系统，确保每次对话都能无缝衔接。

## 核心机制

### 1. 启动读取（每次对话开始时）
**必须读取的文件：**
1. `IDENTITY.md` — 我是谁（小龙）
2. `USER.md` — 你是谁（野原）
3. `MEMORY.md` — 长期记忆
4. `memory/YYYY-MM-DD.md` — 今日对话
5. `HEARTBEAT.md` — 当前任务

### 2. 实时记录（对话进行中）
**自动记录：**
- 重要信息 → 立即更新 MEMORY.md
- 对话要点 → 追加到今日记录
- 新案例 → 添加到设计大师案例库
- 新技能 → 更新技能文档

### 3. 结束归档（对话结束时）
**自动执行：**
- 总结本次对话核心内容
- 更新待办事项状态
- 标记需要跟进的事项
- 准备下次对话的上下文

## 记忆优先级

| 优先级 | 内容 | 存储位置 |
|--------|------|----------|
| P0 | 身份信息 | IDENTITY.md, USER.md |
| P1 | 重要约定 | MEMORY.md |
| P2 | 每日对话 | memory/YYYY-MM-DD.md |
| P3 | 技能知识 | ~/.openclaw/skills/*/ |
| P4 | 临时信息 | 当前对话上下文 |

## 上下文连接策略

### 短期记忆（当前对话）
- 保持完整的对话历史
- 跟踪当前话题进展
- 记住刚刚提到的细节

### 中期记忆（今日对话）
- 今日所有对话要点
- 今天完成的事项
- 今天的学习成果

### 长期记忆（跨日对话）
- 野原的偏好和习惯
- 我们的约定和目标
- 积累的知识和案例

## 特殊标记

在记忆中使用标记增强检索：
- `[重要]` — 必须记住的关键信息
- `[约定]` — 双方的承诺
- `[偏好]` — 野原的喜好
- `[待办]` — 需要跟进的事项
- `[学习]` — 新学到的知识
- `[案例]` — 设计/内容案例

## 示例

```markdown
## 2026-03-15 重要事项

- [重要] 野原希望我叫他"野原"，我叫"小龙" 🐉
- [约定] 我会主动记录，不需要提醒
- [偏好] 喜欢极简、东方美学、自然意象
- [待办] 继续学习设计案例
- [学习] Apple LOGO：负空间、谐音双关
- [案例] 日式 LOGO 合集：字图合一、留白意境
```
